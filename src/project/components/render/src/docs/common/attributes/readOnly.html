readOnly

The readOnly attribute controls whether an input renders in a readonly or updatable state. By default all pages load in either level true (readonly) or level false (updatable) modes. This attribute allows developers to toogle this rendering modes at the field level.

Example

Custom Tag

<E2:Text type="text" viewKey="demo" readOnly="true" />

Example JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Read Only Text Label:",
        "attributes": {
            "for": "demo"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo",
            "type": "text",
            "value": "Read Only Value",
        },
        "readOnly": true
    }
}

For the purposes of this example we are using a <E2:TextField>. For the full support list see below. We are also assuming that in this case, the page loaded in update mode.

As shown in the JSON, by including the readOnly mode attribute on a input JSP tag, an additional input property of readOnly has been created in the JSON. The value the developer specifies the value for the attribute; a value of ture being readonly and a value of false being editable (update mode).

Rendered Examples

This attribute can be set on the following custom tags:

  • Checkbox
  • Date
  • selectBox
  • Text
  • Textarea

Example

RadioGroup Tag

<E2:RadioGroup viewKey="colorValue">
    <E2:Radio label="Blue" value="blue" />
    <E2:Radio label="Red" value="red" />
</E2:RadioGroup>

Example JSON

{
    "type": "radio",
    "template": "inputGroup",
    "legend": "Pick a color",
    "options": [
        {
            "type": "radio",
            "template": "field",
            "label": {
                "text": "Blue",
                "attributes": {
                    "for": "demo_1"
                }
            },
            "input": {
                "attributes": {
                    "name": "demo",
                    "id": "demo_1",
                    "type": "radio",
                    "value": "blue"
                }
            }
        },
        {
            "type": "radio",
            "template": "field",
            "label": {
                "text": "Red",
                "attributes": {
                    "for": "demo_2"
                }
            },
            "input": {
                "attributes": {
                    "name": "demo",
                    "id": "demo_2",
                    "type": "radio",
                    "value": "red",
                    "checked": true
                }
            }
        }
    ],
    "readOnly": true
}

Rendered Examples

For the purposes of this example we are using a <E2:RadioGroup>. For the full support list see below. We are also assuming that in this case, the page loaded in update mode.

Group elements normally provide users with more than one option or value to pick from. When turned into a readOnly field though only the selected values are actually displayed. The value of course being the label text value is normally visible to users in its updatable state.
