accessKey

The accessKey attribute is an accessibility feature found in all browsers. It allows users to jump around page to different elements by using a keyboard shortcut, generally the alt key and a letter designated by the developer. For system and browser specifics key combinations see this WebAIM article.

Examples

Custom Tag

<E2:Text type="text" viewKey="demo" accessKey="G" />

Sample JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Press Alt+G to focus",
        "attributes": {
            "for": "demo4"
        }
    },
    "input": {
        "attributes": {
            "name": "demo4",
            "id": "demo4",
            "type": "text",
            "placeholder": "Press G to get focus",
            "accesskey": "G"
        },
    }
}

For the purposes of this example we are using a <E2:TextField>. For the full support list see below.

To create an accesskey on any supported custom tag, simply call the attribute provide it a letter value. Note that each accesskey used on a page must be unique. When the tag is converted to JSON, the accessKey property should appear inside the attributes property of the primary interactive element. Interactive elements include: buttons, inputs, select boxes; or any other element an end user might click or enter data.

Rendered Examples

For the purposes of this example, press the "G" key, with the correct corresponding system and browser accesskey combination. The browser should focus directly onto this input.

This attribute can be set on the following custom tags:

  • Button
  • Checkbox
  • Date
  • SelectBox
  • Password
  • Radio
  • Text
  • Textarea
