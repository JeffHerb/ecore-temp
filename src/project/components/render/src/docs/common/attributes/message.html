Message Attribute

Provide field level messages to at the field or group level.

Field Message Example

Example JSP Tag

<E2:Text type="text" viewKey="demo" message="<%= errorMessage %>" />

Example JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Change the label visibility",
        "attributes": {
            "for": "demo"
        },
        "visibility": "hidden",
        "hasLayout": false
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo",
            "type": "text",
            "value": "1,000.00"
        }
    },
    "message": {
        "type": "error",
        "text": "This is a error simple message!"
    }
}

For the purposes of this example we are using a <E2:Text>. For the full support list see below.

Rendered Examples

The message allows developers to send message along with inputs. The developer needs to provide the type of message that should be displayed and the text of the message. For more infomation on different message types see the E2:Message tag page. When this is rendered into JSON, the message property appears as part of the root field object.

Supported On:

  • Date
  • selectBox
  • Textarea
  • Text

Field Message Example

Example JSP Tag

<E2:InputGroup legend="Full Name" message="<%= errorMessage %>">
    <E2:Text type="text" viewKey="E_FIRST_NAME" visibility="HIDE_LABEL_NO_SPACE" />
    <E2:Text type="text" viewKey="E_MIDDLE_INITIAL" visibility="HIDE_LABEL_NO_SPACE" />
    <E2:Text type="text" viewKey="E_LAST_NAME" visibility="HIDE_LABEL_NO_SPACE" />
</E2:InputGroup>

Example JSON

{
    "type": "inputs",
    "template": "inputGroup",
    "legend": "Full Name:",
    "options": [
        {
            "type": "text",
            "template": "field",
            "label": {
                "text": "First Name",
                "attributes": {
                    "for": "E_FIRST_NAME"
                },
                "visibility": "hidden",
                "hasLayout": false
            },
            "input": {
                "attributes": {
                    "name": "E_FIRST_NAME",
                    "id": "E_FIRST_NAME",
                    "type": "text",
                    "value": "",
                }
            }
        },
        {
            "type": "text",
            "template": "field",
            "label": {
                "text": "Middle Initial",
                "attributes": {
                    "for": "E_MIDDLE_INITIAL"
                },
                "visibility": "hidden",
                "hasLayout": false
            },
            "input": {
                "attributes": {
                    "name": "E_MIDDLE_INITIAL",
                    "id": "E_MIDDLE_INITIAL",
                    "type": "text",
                    "value": "",
                    "size": 1
                }
            }
        },
        {
            "type": "text",
            "template": "field",
            "label": {
                "text": "Last Name",
                "attributes": {
                    "for": "E_LAST_NAME"
                },
                "visibility": "hidden",
                "hasLayout": false
            },
            "input": {
                "attributes": {
                    "name": "E_LAST_NAME",
                    "id": "E_LAST_NAME",
                    "type": "text",
                    "value": "",
                }
            }
        }
    ],
    "message": {
        "type": "error",
        "text": "This is a error simple message!"
    }
}

For the purposes of this example we are using a <E2:InputGroup>. For the full support list see below.

Rendered Examples

Similar to the example above. By adding the message attribute on the group tag (in this case E2:InputGroup) any error messages found will appear under the group in question. In the JSON, the message attribute should be found at the root level of the input group object.

Supported On:

  • CheckBoxGrouo
  • InputGroup
  • RadioGroup
