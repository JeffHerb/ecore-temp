required

The required attribute is used to signify that said input must be filled in, in order for the user to submit a specific web form.

Example

Example JSP Tag

<E2:Text type="text" viewKey="demo" required="true" />

Example JSON

{
    {
        "type": "text",
        "template": "field",
        "label": {
            "text": "Required Text Label",
            "attributes": {
                "for": "demo"
            }
        },
        "input": {
            "attributes": {
                "name": "demo",
                "id": "demo",
                "type": "text",
                "aria-required": "true"
            },
            "required": true
        }
    }
}

For the purposes of this example we are using a text field. For the full support list see below. We are also assuming that in this case, the page loaded in update mode.

Rendered Examples

The required attribute used on a custom tag actual means that an additional sub property of required will be added to the input JSON object. This will also add a new property to the input attributes object of "aria-required" and it must have a value of true.

This attribute can be set on the following custom tags:

  • Date
  • SelectBox
  • Textarea
  • Text

Required Group

Example JSP Tag

<E2:InputGroup legend="Name" required="true">
    <E2:Text type="text" viewKey="fname" required="true" />
    <E2:Text type="text" viewKey="mname" />
    <E2:Text type="text" viewKey="lname" required="true" />
</E2:InputGroup>

Rendered Examples

The required attribute does not need to be defined on input groups. This is because required asterisks will only appear on an input group when the input contents are all or partially marked as required and the individual input labels are marked as hidden (the exception being for radio and checkboxes). In the example below, the first and the last name inputs are marked as required fields. The middle initial is not required. But since all three have hidden labels, it is assumed that the individual input do not require additional explination and that the require asterisk can safely be placed on the input group legend.

This attribute can be set on the following custom tags:

  • CheckBoxGroup
  • InputGroup
  • RadioGroup
