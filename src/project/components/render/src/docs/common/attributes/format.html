Format Attribute

In Review

Masks are specific ways a particular data should appear when displayed.

  • Dates
  • Numbers

Formats can be used on any type of output based tag below:

  • Date
  • Table Column
  • Text

Date Masks

Example JSP Tag

<E2:Date viewKey="date_demo" format="MM/dd/yyyy" /> 
- or -
<E2:Text control="date" type="date" viewKey="date_text_demo" format="MM/dd/yyyy" />

Example JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Last date of the year",
        "attributes": {
            "for": "demo"
        },
        "visibility": "hidden",
        "hasLayout": false
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo",
            "type": "text",
            "value": "12/31/2015"
        }
    }
}

The date format can be used on on either the E2:Date or the E2:Text with the type attribute set to date. Formatting is done on the server side, so when developers pass in a format, the string value needs to meet the time date specifications provided in Java. For a full list of date time masks please consult the Oracle documentation.

Rendered Examples

Formats provide no addition markup at the JSON level. Existing values if any are already converted on and server in the inputs Value attribute.

Date Masks

Example JSP Tag

<E2:Text viewKey="text1" format="quantity" />

Example JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Number Value with formatting",
        "attributes": {
            "for": "demo2"
        }
    },
    "input": {
        "attributes": {
            "name": "demo2",
            "id": "demo2",
            "type": "text",
            "value": "1,235,897.00"
        }
    }
}

Number formatting is based on the Java number and currancy format spec. For more specific information on different formats, please reference the Oracle documentation. In most cases providing the value of either quanity or currancy is enough to outbut the desired proper formatting values.
