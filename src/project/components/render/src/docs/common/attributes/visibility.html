Visibility Attribute

Changes the rendering state of part or all of a custom tags contents. These visibility settings can be toggled via the page more or event at the field level. For more information please see the different visibility states below.

  • HIDE_ON_READ
  • HIDE_ON_UPDATE
  • HIDE_LABEL
  • HIDE_LABEL_NO_SPACE
  • HIDE_LABEL_YES_SPACE
  • HIDE_COLUMN

HIDE_ON_READ

This will completely hide the contents of a custom tag. If a developer lists this as a visibility setting, when the page is in read mode, the JSON that would normally be included for this custom tag will be omitted.

This attribute can be set on the following custom tags:

  • Button
  • ButtonGroup
  • Checkbox
  • CheckboxGroup
  • Date
  • Group
  • InputGroup
  • RadioGroup
  • SelectBox
  • TabNavButton
  • TabNavLink
  • Textarea
  • Text

HIDE_ON_UPDATE

This will completely hide the contents of a custom tag. If a developer lists this as a visibility setting, when the page is in update mode, the JSON that would normally be included for this custom tag will be omitted.

This attribute can be set on the following custom tags:

  • Button
  • ButtonGroup
  • Checkbox
  • CheckboxGroup
  • Date
  • Group
  • InputGroup
  • RadioGroup
  • SelectBox
  • TabNavButton
  • TabNavLink
  • Textarea
  • Text

HIDE_LABEL

This will cause the omition of the Label element for any input based custom component.

This attribute can be set on the following custom tags:

  • Date
  • SelectBox
  • Textarea
  • Text

HIDE_LABEL_NO_SPACE

Example JSP Tag

<E2:Text type="text" viewKey="demo" visibility="HIDE_LABEL_NO_SPACE" />

Example JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Change the label visibility",
        "attributes": {
            "for": "demo"
        },
        "visibility": "hidden",
        "hasLayout": false
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo",
            "type": "text",
            "value": "1,000.00"
        }
    }
}

For the purposes of this example we are using a <E2:TextField>. For the full support list see below.

Rendered Examples

In the example above, the visibility attribute was set to HIDE_LABEL. This means that no outter div with the class of and the value of cui-label will be wrapping the inputs label. An additional class is also added label tag with the value cui-hide-from-screen. As demonstrated below, this causes the input to appear all the way to the left most side of the example container.

This attribute can be set on the following custom tags:

  • Date
  • SelectBox
  • Textarea
  • Text

HIDE_COLUMN

Custom JSP Tag

<E2:Table data="<% arrayList %>">
    <E2:TableHead>
        <E2:TableRow>
            <E2:TableColumn name="Col1" property="Col1"/>
            <E2:TableColumn name="Col2" property="Col2"/>
            <E2:TableColumn name="Col3" property="Col3"/>
            <E2:TableColumn name="Col4" property="Col4" visibility="HIDE_COLUMN" />
            <E2:TableColumn name="Col5" property="Col5" visibility="HIDE_COLUMN" />
        <E2:TableRow />
    <E2:TableHead />
<E2:Table />

Example JSON

{
    "template": "table",
    "head": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "text": "Row Index",
                        "skip": "soft"
                    },
                    {
                        "type": "header",
                        "text": "1"
                    },
                    {
                        "type": "header",
                        "text": "2"
                    },
                    {
                        "type": "header",
                        "text": "3"
                    },
                    {
                        "type": "header",
                        "text": "4",
                        "visibility": "hidden"
                    },
                    {
                        "type": "header",
                        "text": "5",
                        "visibility": "hidden"
                    }
                ]
            }
        ]
    },
    "body": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "text": "1"
                    },
                    {
                        "text": "1,1"
                    },
                    {
                        "text": "1,2"
                    },
                    {
                        "text": "1,3"
                    },
                    {
                        "text": "1,4",
                        "visibility": "hidden"
                    },
                    {
                        "text": "1,5",
                        "visibility": "hidden"
                    }
                ]
            },
            ... remaining data rows omitted ...
        ]
    }
}

Rendered Examples

In the example above, the visibility attribute was set to HIDE_COLUMN on the last two table columns. What this does on the the JSON side is add the property of visibility with the value of hidden to the column object. This will cause the render to include the columns in the DOM markup, but will add a class to cause the columns to not render visaully to the screen.

This attribute can be set on the following custom tags:

  • TableColumn
