ButtonRow and ButtonColumn

New in E2

To create a group of buttons the developer must organize the buttons using the <E2:ButtonRow> and <E2:ButtonCol> tags. This is similar to how inout fields are laid out in a grid using the <E2:Row> and <E2:Col> tags.

At minimum, a group of buttons must have one row containing one column. This may be expanded to include two columns. The buttons must go inside a <E2:ButtonCol> tag.

Button Rows and Columns

<E2:ButtonRow>
    <E2:ButtonCol size="half">
           <E2:Button viewKey="button1" />
           <E2:Button viewKey="button2" />
    </E2:ButtonCol>
</E2:ButtonRow>

Sample JSON

{
    "type": "row",
    "template": "buttonGroup",
    "contents": [
        {
            "type": "column",
            "template": "buttonGroup",
            "contents": [
                {
                    "type": "button",
                    "template": "field",
                    "input": {
                        "attributes": {
                            "id": "cancel",
                            "name": "Cancel",
                        },
                        "text": "Cancel"
                    }
                },
                {
                    "type": "button",
                    "template": "field",
                    "input": {
                        "attributes": {
                            "id": "update",
                            "name": "update",
                        },
                        "text": "Update",
                        "primary": true
                    }
                }
            ]
        }
    ]
}

While the Java developer uses the row-and-column layout approach, the JSON sets the template property to call the buttonGroup template. This adds necessary styles only used with button rows. <E2:ButtonCol> follows the same size controls used by <E2:Col>.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
align Boolean No Sets the buttons screen alignment. Acceptable Values
  • Left
  • Center
  • Right
htmlAttributes String No Adds HTML attributes to the rendered <div> container created for either the ButtonRow or the ButtonCol (more details)
visibility String No Controls the visibility state of a particular element (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE

Align

<E2:ButtonRow>
    <E2:ButtonCol size="half" align=“right”>
           <E2:Button viewKey="button3" />
           <E2:Button viewKey="button4" />
    </E2:ButtonCol>
</E2:ButtonRow>

Sample JSON

{
    "type": "row",
    "template": "buttonGroup",
    "contents": [
        {
            "type": "column",
            "template": "buttonGroup",
            "align": "right",
            "contents": [
                {
                    "type": "button",
                    "template": "field",
                    "input": {
                        "attributes": {
                            "id": "cancel",
                            "name": "Cancel",
                        },
                        "text": "Cancel"
                    }
                },
                {
                    "type": "button",
                    "template": "field",
                    "input": {
                        "attributes": {
                            "id": "update",
                            "name": "update",
                        },
                        "text": "Update",
                        "primary": true
                    }
                }
            ]
        }
    ]
}

ButtonCol has one additional attribute of align which is used to align its buttons to the right or left. In the JSON, the additional property of align is created on the column object and it has the value of true.
