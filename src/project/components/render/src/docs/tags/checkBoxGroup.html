CheckBoxGroup Custom Tag

New in E2

The <E2:CheckBoxGroup> tag combines a series of <E2:CheckBox> tags into a logical group. This is primarily used for check lists ("select all that apply", etc). Each group will have a single legend in addition to the check boxes which acts as a label for the entire group.

Examples

Custom Tag

<E2:CheckBoxGroup legend="Choose a color">
    <E2:CheckBox viewKey="E_CHECK_BLUE" value="blue" />
    <E2:CheckBox viewKey="E_CHECK_RED" value="red" />
</E2:CheckBoxGroup>

Sample JSON

{
    "type": "checkbox",
    "template": "inputGroup",
    "legend": "Choose a color",
    "options": [
        {
            "type": "checkbox",
            "template": "field",
            "label": {
                "text": "Blue",
                "attributes": {
                    "for": "E_CHECK_BLUE"
                }
            },
            "input": {
                "attributes": {
                    "name": "E_CHECK_BLUE",
                    "id": "E_CHECK_BLUE",
                    "type": "checkbox",
                    "value": "blue"
                }
            }
        },
        {
            "type": "checkbox",
            "template": "field",
            "label": {
                "text": "Red",
                "attributes": {
                    "for": "E_CHECK_RED"
                }
            },
            "input": {
                "attributes": {
                    "name": "E_CHECK_RED",
                    "id": "E_CHECK_RED",
                    "type": "checkbox",
                    "value": "red"
                }
            }
        }
    ]
}

When this JSP tag is used the JSON should call the <E2:InputGroup> template and the type property should be set to checkbox. One major difference when using the <E2:InputGroup> template compared to other container-type tags is that it uses the options property for all its children instead of the contents property. This is because inputGroup template will handle its children via templates instead of the standard render logic.

Developers need to set the legend attribute for this tag. All check box-specific settings are handled by those individual custom tags.
