Form Custom Tag

All fields on a page are contained inside form tags.

Examples

Custom Tag

<E2:Form>
</E2:Form>

Sample JSON

{
    "type": "form",
    "attributes": {
        "name": "form1",
        "method": "POST",
        "action": "http://www.ny.gov"
    },
    contents: []
}

<E2:Form> tags are container elements that developers should use to wrap all of their input fields. The only required attribute is the name attribute. Since a page can have more than one form on it, the form names must be unique.

While Java developers only need to name the form on the page, the JSON append additional attributes needed for the form to properly function. For example, all forms need method and action attributes. These attributes should be handled by framework and developer can override using the htmlAttributes attribute.

Default example

By default forms render no visible content. Use the browser development tool on the box below to see the rendered output.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
name String Yes Specifies the name and id attributes for the <form> HTML element. If this is not set it will default to form1.
htmlAttributes String No Adds HTML attributes to the rendered <div> container (more details)
eventScripts String No Any event-based JavaScript that needs to be executed. The string must include both event type and function name (more details)
