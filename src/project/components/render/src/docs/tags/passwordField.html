Password Custom Tag

The Password is a masked version of the standard text input. Both passwordField and the <E2:Text> custom tags share many of the same custom tag attributes. Below only the attributes specific to passwordField tag will be explained.

Examples

Custom Tag

<E2:Password viewKey="demo" />

Sample JSON

{
    "type": "password",
    "template": "field",
    "label": {
        "text": "Password",
        "attributes": {
            "for": "login-password"
        }
    },
    "input": {
        "attributes": {
            "name": "login-password",
            "id": "login-password",
            "type": "password",
            "value": "Password1"
        }
    }
}

To define any PasswordField input, developer need to implement the JSP tag and specify the viewKey the same way the would for a common <E2:Text> tag.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
type String Yes Defines the HTML 5 input type that should be created. Please see the acceptable type list above.
viewKey String Yes A registered view key (previously called pageElementName)
required Boolean No Indicates whether the field must be filled in (more details)
readOnky Boolean No Indicates whether the input should be read-only (more details)
title String No Sets the title attribute of the input element.
eventScript String No Any event-based JavaScript that needs to be executed. The string must include both event type and function name (more details)
accessKey String No Shortcut key to set focus to the field (more details)
visibility String No Used to control the visibility state of a particular input and/or its label or parts (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE
  • HIDE_LABEL
  • HIDE_LABEL_NO_SPACE
  • HIDE_LABEL_YES_SPACE
labelHtmlAttributes String No Adds HTML attributes to the rendered <label> tags (more details)
inputHtmlAttributes String No Adds HTML attributes to the rendered <input> tags (more details)
property Object No Java object attribute from which value is populated. It is the responsibility of the tag or its parent tag to make the object available to the page context.
rsMapping String No Used to map this field to result set. Please see the ResultSet custom tag documentation for details.
message hashmap No Provides a hashmap of a specific field level message to an input (more details).
