RadioGroup Custom Tag

New in E2

The <E2:RadioGroup> tag is used to help structure

Examples

Custom Tag

Manual list creation with tags

<E2:RadioGroup viewKey="colorValue">
    <E2:Radio label="Blue" value="blue" />
    <E2:Radio label="Red" value="red" />
</E2:RadioGroup>

Provided source list

<E2:RadioGroup viewKey="colorValue" options="<%= radioKeyValues  %>">

Sample JSON

{
    "type": "radio",
    "template": "inputGroup",
    "legend": "Pick a color",
    "options": [
        {
            "type": "radio",
            "template": "field",
            "label": {
                "text": "Blue",
                "attributes": {
                    "for": "demo_1"
                }
            },
            "input": {
                "attributes": {
                    "name": "demo",
                    "id": "demo_1",
                    "type": "radio",
                    "value": "blue"
                }
            }
        },
        {
            "type": "radio",
            "template": "field",
            "label": {
                "text": "Red",
                "attributes": {
                    "for": "demo_2"
                }
            },
            "input": {
                "attributes": {
                    "name": "demo",
                    "id": "demo_2",
                    "type": "radio",
                    "value": "red"
                }
            }
        }
    ]
}

RadioGroups is a very special component as developers can define this tag in two different ways. The first one is the manual process, this is where developers declare each E2:Radio input by hand. Or developers can pass in the a key value list of every radio button option via the options attribute.

Another signification change is developer need to list the viewKey attribute on the group itself. This is because the viewKey label will become the legend for the RadioGroup instead of the label for the individual options.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
viewKey String Yes The registered viewkey
htmlAttributes String No Adds HTML attributes to the rendered <fieldset> container (more details)
visibility String No Used to control the visibility state of a particular input and/or its label or parts (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE
flip Boolean No Manualy flip the order of Input Label to Label Input.
message hashmap No Provides a hashmap of a specific field level message to an input (more details).
layout String No Controls the way the elements are listed on the page. Developers can only define either vertical or horizontal layout. The default layout is horizontal.

Custom Tag

<E2:RadioGroup viewKey="colorValue" options="<%=radioKeyValues%>" layout="vertical">

Sample JSON

{
    "type": "radio",
    "template": "inputGroup",
    "legend": "Pick a color",
    "options": [
        ... Ommited ...
    ],
    "layout: "vertical"
}

By default all E2:RadioGroups render in as a horizontal list. In order to toggle the list to appear vertically developers need to add the attribute of layout with the value of vertical.
