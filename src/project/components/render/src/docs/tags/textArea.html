Textarea Custom Tag

The Textarea field is a carbon copy of your standard text JSP tag. The only difference is text area are used for large amounts of text that can appear across multiple lines.

Examples

Custom Tag

<E2:TextArea viewKey="demo" />

Sample JSON

{
    "type": "textarea",
    "template": "field",
    "label": {
        "text": "When to use TextArea's",
        "attributes": {
            "for": "date-demo"
        }
    },
    "input": {
        "attributes": {
            "name": "date-demo",
            "id": "date-demo",
            "type": "textarea",
            "value": "This type of tag is for where you have a large amount of information. Something like a paragraph or simply a few sentances."
        }
    }
}

To define any TextArea input, developer need to implement the JSP tag and specify the viewKey the same way the would for a common <E2:Text> tag. Unlike the standard <E2:Text> the type attribute doesn't have to be defined as the tag itself implys the component type.

value String No Default value of the input.
required Boolean No Indicates whether the field must be filled in (more details)
readOnly Boolean No Indicates whether the input should be read-only (more details)
rows Integer No Number of rows of text to display by default
title String No Sets the title attribute of the input element.
eventScript String No Any event-based JavaScript that needs to be executed. The string must include both event type and function name (more details)
accessKey String No Shortcut key to set focus to the field (more details)
visibility String No Used to control the visibility state of a particular input and/or its label or parts (more details)

  • HIDE_ON_READ
  • HIDE_ON_UPDATE

  • HIDE_LABEL_YES_SPACE

property Object No Java object attribute from which value is populated. It is the responsibility of the tag or its parent tag to make the object available to the page context.

message hashmap No Provides a hashmap of a specific field level message to an input (more details).

Examples

Custom Tag

<E2:TextArea type="text" viewKey="demo2" rows="10" />

Rows JSON

{
    "type": "textarea",
    "template": "field",
    "label": {
        "text": "10 row example",
        "attributes": {
            "for": "date-demo2"
        }
    },
    "input": {
        "attributes": {
            "name": "date-demo2",
            "id": "date-demo2",
            "type": "textarea",
            "rows": 10
        }
    }
}

Textareas support the ability to change the elements hieght without using style. To do this, developers need to declare the rows attribute. This sets the number of rows a textarea will display before forcing the element to start scrolling on the vertical axis. Since rows is a native attribute it will appear inside of the input attribute object when generated.
