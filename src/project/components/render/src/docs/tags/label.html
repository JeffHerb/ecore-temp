Label Custom Tag

In Review

The <E2:Label> is used when you want to change the default way that <label> and <inputs> HTML elements are displayed.

Examples

Custom Tag

<E2:Label viewKey="E_FIELD" />

Sample JSON

{
    "template": "field",
    "label": {
        "text": "Separate Label",
        "attributes": {
            "for": "E_FIELD"
        }
    }
}

Normally <label> elements are generated and laid out automatically by using just a field custom tag. The advantage is that developers don't need to worry about the layout or maintaining extra code.

However in some situations it may be required to display inputs and their labels in a non-standard way. To do this, use separate <E2:Label> and field custom tags as described below.

Default example

For example, suppose we must separate a label from its text input by placing each element into their own column. Both custom tags must use the same view key to ensure they both receive the correct label text and other attributes. The <E2:Text> on the other hand should also include visibility="HIDE_LABEL" since the corresponding label is being created separately.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
viewKey String Yes A registered view key (previously called pageElementName)
htmlAttributes String No Adds HTML attributes to the rendered <label> element (more details)
