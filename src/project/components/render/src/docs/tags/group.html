Custom Tag

<E2:Group>
    <E2:GroupHeader title="Section Title">
    </E2:GroupHeader>
    ... Group Contents ...
</E2:Group>

Sample JSON

{
    "type": "section",
    "contents": [
        {
            "type": "header",
            "template": "group",
            "title": "Section Title"
        },
        {
            "type": "row",
            "template": "grid",
            "contents": [
                {
                    "type": "column",
                    "template": "grid",
                    "contents": [
                        {
                            "type": "p",
                            "text": "Example group container"
                        }
                    ]
                }
            ]
        },
    ]
}

To create a default group container, developers simply need to use the <E2:Group> tag. This tag itself doesn't have any special property, it is just a placeholder for additional contents. The first tag inside any Group should be the <E2:GroupHeader> this tag helps define the blue header section found below. Note in the Default JSON example, that the <E2:Group> tag only defines the tag and contents properties.

The E2:GroupHeader tag has a few key properties, the first being the group title. Group titles are set using the title attribute on the group tag. By default, group controls found on the far left of the group header are visible.

Header Columns

Custom Tag

<E2:Group>
    <E2:GroupHeader title="Group Title">
        <E2:GroupHeaderCol text="Last Checked: 1/1/2015" />
        <E2:GroupHeaderCol>
            <E2:CheckBox viewKey="retainData" />
        </E2:GroupHeaderCol>
    </E2:GroupHeader>
</E2:Group>

Sample JSON

{
    "type": "section",
    "contents": [
        {
            "type": "header",
            "template": "group",
            "title": "Group Title",
            "columns": [
                {
                    "text": "Last Checked: 1/1/2015"
                },
                {
                    "type": "checkbox",
                    "template": "field",
                    "label": {
                        "text": "Retain Data",
                        "attributes": {
                            "for": "RETAIN_DATA_CHECK"
                        }
                    },
                    "input": {
                        "attributes": {
                            "name": "RETAIN_DATA_CHECK",
                            "id": "RETAIN_DATA_CHECK",
                            "type": "checkbox",
                            "value": "y"
                        }
                    }
                }
            ]
        },
        {
            "type": "row",
            "template": "grid",
            "contents": [
                {
                    "type": "column",
                    "template": "grid",
                    "contents": [
                        {
                            "type": "p",
                            "text": "Example group container"
                        }
                    ]
                }
            ]
        },
    ]
}

To create additional columns in a group header, developers should use the E2:GroupHeaderCol tag inside of the E2:GroupHeader tag. Depending on the contents, developer can create single line tags or nest additional custom tags between the standard open and closing tags.

Whenever a group header includes the E2:GroupHeaderCol the header JSON should add a columns array property, and create an object for each of the additional tags found within. For simple one line text columns, the new sub object can have just a text property and the developer entered value. For more complex contents, the new sub-object JSON should followin the standard JSON definition for the request tag within.

Header Indicators

Custom Tag

<E2:Group>
    <E2:GroupHeader title="Group Title">
        <E2:GroupIndicators>
            <E2:TabNavButton viewKey="tabButton1" text="Claim History" />
        </E2:GroupIndicators>
    </E2:GroupHeader>
</E2:Group>

Sample JSON

{
    "type": "header",
    "template": "group",
    "title": "Group Title",
    "indicators": [
        {
            "type": "button",
            "template": "tabNavigation",
            "attributes": {
                "tabindex": 1,
                "title": "Legal Information"
            },
            "icon": "legal",
            "text": "Legal Information",
        }
    ]
}

Some group have additional properties and functionalities based on the status of a given account or groups specific contents. Because of this, additional items may appear directly next to the group titile. Too add these items Developers can add the E2:GroupIndicators tag. The content of this tag is normally a E2:TabNavButton and based on the security and page mode the element could render as a button or an inline-icon.

Whenever the E2:GroupIndicators is in use, the header object should contain the additional array property of indicators. The contents of this array should be the output of the individual custom tags found within.

hideControls

Custom Tag

<E2:Group>
    <E2:GroupHeader title="Group Title" hideControls="true" />
</E2:Group>

Sample JSON

{
    "type": "header",
    "template": "group",
    "title": "Group Title",
    "hideControls": true
}

By default all group controls (buttons located to the right most side of group title bar) are set to visible by default. If they need to be toggled off, on the GroupHeader tag, developers need to implement the hideControls attribute, and set it to true.
