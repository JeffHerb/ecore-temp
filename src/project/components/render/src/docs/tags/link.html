Link Custom Tag

The E2:Link is the generic way to create an anchor control that can be used for event bindings or navigational peices. This tag combinds the functionality formerally found in:

  • Common Web Links
  • ET:TabNavLink
  • ET:ExternalLinkButton

Examples

Custom Tag

<E2:Link href="http://www.google.com" text="Google" />

Sample JSON

{
    "template": "link",
    "attributes": {
        "href": "https://www.google.com"
    },
    "text": "Google"
}

By the E2:Link tag acts link any other anchor tag. By providing it a href attribute and a text attribute will cause the tag to render as a standard web hyperlink. If the link requires additional functionality or opens tabSet pages, then a viewKey should be provided and the link needs to be registered with the registration tool.

When a link is turned to JSON, the tag needs to include a reference to the template property and should have the value of link. The display text or the JSP text attributer is turned into the text property. Lastly the attributes property should be included and a copy of the all the link specific attributes should be include. On a standard web link, the only required property is the href property.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
href or viewKey String No Used to define where the link tag is going. If the href attribute is defined, its assumed the link goes a page that can be reached via a normal http request. If a viewKey is defined instead, the anchor tag will used the information provided by the registration tool.
text String No Text that should be used when rendering the link.
title String No Sets the title attribute of the input element.
disabledText String No The text displayed when the element is rendered in readOnly or disabled mode. If this attribute is missing the normal text is used.
preSubmitScript String No Name and arguments to a function that is execute "onclick", but before the form has been submitted (more details)
eventScript String No List of event-based JavaScript associated with the field. The string must include both the event type and function name (more details)
requestedMode Boolean No Indicates whether the input should be read-only or updatable mode. See the disabled example below on how requestedMode "1" or readonly should be treated.
visibility String No Used to control the visibility state of a particular input and/or its label or parts (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE
htmlAttributes String No Adds HTML attributes to the rendered wrapper <div> container (more details)
