InputGroup Custom Tag

New in E2

The <E2:InputGroup> tag is a semantic way to combine related inputs together into a single grouping. This is useful for most inputs, such as text fields, but for radio buttons and check boxes see the <E2:RadioGroup> and <E2:CheckBoxGroup> tags, respectively.

Examples

Custom Tag

<E2:InputGroup legend="Full Name">
    <E2:Text type="text" viewKey="E_FIRST_NAME" visibility="HIDE_LABEL_NO_SPACE" />
    <E2:Text type="text" viewKey="E_MIDDLE_INITIAL" visibility="HIDE_LABEL_NO_SPACE" />
    <E2:Text type="text" viewKey="E_LAST_NAME" visibility="HIDE_LABEL_NO_SPACE" />
</E2:InputGroup>

                "attributes": {

The only required attribute on the custom tag is legend. This string acts as a label for the explain the contents inside. All input-specific settings are handled by those individual custom tags.

Rendered examples

If you look at the example you will notice that the text fields all contain the visibility attribute. Since the contents of the 3 fields are explained by the legend, it is safe to accessibly hide each input's label.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
legend String Yes A label that applies to the entire collection of fields
htmlAttributes String No Adds HTML attributes to the rendered <fieldset> container (more details)
message hashmap No Provides a hashmap of a specific field level message to an input (more details).
visibility String No Used to control the visibility state of a particular input and/or its label or parts (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE
