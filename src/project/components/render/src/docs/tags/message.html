Message Custom Tag

In Review

The <E2:Message> tag is used to style messages for information, error, warning and success purposes.

Examples

Custom Tag

<E2:Message text="This is a simple message!" />
<E2:Message text="This is a simple error message!" type="error" />
<E2:Message text="This is a simple success message!" type="success" />
<E2:Message text="This is a simple warning message!" type="warning" />

Sample JSON

{
    "type": "info",
    "template": "message",
    "text": "This is a simple message!"
}

Messages are a nice way to give a specific piece of information more prominent on a page. By default, all message will appear as blue informational messages. To change this, define the type attribute with one of the acceptable values listed in the table below.

Default example

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
text String Yes Text that will be displayed.
list Key-Value Yes Key value pair for each of the messages that should appear in the message block.
type String No dentifies the type of style that message will appear as. If this is not given the type will default to info. Accepts only the following values
  • info - Informational - Blue
  • warn - Warnings - Orange
  • error - Error - Red
  • success - Success - Green
htmlAttributes String No Adds HTML attributes to the rendered <div> container (more details)

Examples

Custom Tag

<E2:Message list="<%$= listOfMessages %>" />

Sample JSON

{
    "template": "message",
    "list": [
        {
            "type": "info",
            "text": "This is a simple info message!"
        },
        {
            "type": "error",
            "text": "This is a simple error message!"
        },
        {
            "type": "success",
            "text": "This is a simple success message!"
        },
        {
            "type": "warn",
            "text": "This is a simple warning message!"
        }
    ]
}

If there are a list of messages that are related and need to be dislplayed, developers can pass in an array of object. For each message the developer must define a type and text value to use.
