Tables Column Custom Tag

In Review

The <E2:TableColumn> defines the type of column data (header or data) and the whether it should be rendered or not.

Examples

Custom Tag

<E2:Table>
    <E2:TableColumn dataType="alpha" label="Header" />
    <E2:TableColumn dataType="alpha" label="Header" />
</E2:Table>

Sample JSON

{
    "template": "table",
    "head": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "text": "Header",
                        "attribute": {
                            "data-type": "alpha"
                        }
                    },
                    {
                        "type": "header",
                        "text": "Header",
                        "attribute": {
                            "data-type": "alpha"
                        }
                    }
                ]
            }
        ]
    },
    "body": {
        "rows": [
            {
                "columns": [
                    {
                        "text": "Value"
                    },
                    {
                        "text": "Value"
                    }
                ]
            }
        ]
    }
}

When defining columns, the E2:TableColumn makes assumptions and provided hooks that allow developer to control a columns content and context based on different rendering nead. In its most basic form when creating a column developers need to define the minimal requirements of label, which will be the table column headers visual label. dataType which explains the type of data that the column should contain, or be treaded as when its rendered in the table body (tbody). Lastly, property which represents the basic mapping to the tables defined data attribute collection.

When converted to JSON, columns may appear slightly different depending on the rendered section. In the example above, note how the label attribute changes to text when it is found in the head object. Also the additional type propetry. This property simply toggles the table cell to render as a th table header instead of the default td table data cell used mainly in the bottom. The biggest thing to note, the attributes object found out this level directly affect the table cell itself unless the contents contain another set of JSON markup for a different type or renderable element. This can be seen when looking at the dataType JSP attributed is added turned into the data-type HTML attribute that only appears in the table header (thead) section.

At times its possible that developes might need to place more than text or a control inside of a table cell. When this happens the cell definition changes slightly to include the contents array property. In the example below it show a "Select all" checkbox header cell. Since the text still needs to be rendered with the inclusion of the the actual control, the base sell definition has been moved to the contents approach. 

{
    "head": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "attribute": {
                            "data-type": "alpha"
                        },
                        "contents": [
                            {
                                "text": "Select All"
                            },
                            {
                               "type": "checkbox",
                                "template":"field",
                                "label": {
                                    "text": "Select all rows",
                                    "attributes": {
                                        "for": "multi-selection_all"
                                    },
                                    "visibility": "hidden",
                                    "hasLayout": false
                                },
                                "input": {
                                    "attributes": {
                                        "type": "checkbox",
                                        "id": "multi-selection_all",
                                        "name": "multi-selection_all",
                                        "value": "all"
                                    }
                                }
                            }
                        ]
                    },
                ]
            }
        ]
    }
}

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
dataType String No Defines the type of data that will be found in the column. Currently supported data types include:
  • alpha
  • date
  • numeric
label String No The visual or screen readable text that explains the contents of the column. This attribute only works when it appears inside of the <E2:TableHead> tag.
property String No Specifies the getter method that needs to be invoked on the data object to fill the columns contents. It should be a field/property name ion the data/domain oject. Column tag preforms the following operations if property is specified.
  1. Column tag invokes `getXxx` method on the date object where `xxx` ia rhw apwxidiws property. The returned value will fill the columns contents. In the JSON this becomes the text property.
  2. If the above step fails because the method `getXxx` does not exist, it will try to invoke the `isXxx` method to populate the column with the returned value.
  3. If the above step fails because the method `isXxx` does not exist, it would try to access the property/fields drirteclty and populate the column accordingly.
  4. If all the above mentioned methods fail, throw a runtime exception.
hideLabel Boolean No Indicate that the header label for a column should be hidden visually on the table but still be accessible to screen readers.
visibility String No Used to control the visibility state of a particular column (more details).
Acceptable Options:
  • HIDE_COLUMN
style String No Add specific style to a element or a group of elements (more details).
format String No Describes how data should be appear when rendered to the screen. For more details.
sort Boolean No Indicate that this column can sorted on click.
aggregate String No Used to create footer rows that that will aggregate its value from the body columns specified above. Supported Aggregates:
  • Sum - Totals all of the columns into a single sum
  • Average - Create an average amount based on the number of rows

Data Type Attribute

Custom Tag

<E2:Table data="<%= arrayList3x3 %>">
    <E2:TableColumn label="Row Index" />
    <E2:TableColumn dataType="alpha" label="1" />
    <E2:TableColumn dataType="alpha" label="2" />
    <E2:TableColumn dataType="alpha" label="3" />
</E2:Table>

Every column defined in the <E2:TableHead> can also declare its own data type. This additional information is used by interactive components like sort to provide clientside interactive functionality.

Sample JSON

{
    "head": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "text": "Username"
                    },
                    {
                        "type": "header",
                        "text": "Username",
                        "attribute": {
                            "data-type": "alpha"
                        }
                    },
                    {
                        "type": "header",
                        "text": "Username",
                        "attribute": {
                            "data-type": "alpha"
                        }
                    },
                    {
                        "type": "header",
                        "text": "Action",
                        "attribute": {
                            "data-type": "alpha"
                        }
                    }
                ]
            }
        ]
    }
}

When type is converted to JSON it will appear inside the columns object as a sub-property of the columns attributes object. The property name will be data-type and its value will be based on the tables contents. Special columns that contain controls such as buttons or inputs do not need this attribute defined.

Hide Label Attribute

Custom Tag

<E2:Table data="<%= selectableData %>" id="hideLableTable">
    <E2:TableColumn dataType="alpha" label="First Name" property="fname"  />
    <E2:TableColumn dataType="alpha" label="Last Name" property="lname" />
    <E2:TableColumn dataType="alpha" label="Address" property="address" />
    <E2:TableColumn dataType="alpha" label="Address Issue" property="address" style="min-width" hideLabel="true" />
</E2:Table>

The hideLabel attribute is used to accessibiliy hide header label text. This attribute will only affect table columns located in the <E2:TableHead>.

Sample JSON

{
    "template": "table",
    "caption": "Generic Table",
    "head": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "text": "First Name",
                        "attributes": {
                            "data-type": "alpha"
                        }
                    },
                    {
                        "type": "header",
                        "text": "Last Name",
                        "attributes": {
                            "data-type": "alpha"
                        }
                    },
                    {
                        "type": "header",
                        "text": "Address",
                        "attributes": {
                            "data-type": "alpha"
                        }
                    },
                ]
            }
        ]
    },
    "body": {
        ... Omitted Body Content ...
    }
}

When this attribute is used, columns found inside of the head object will have the additional hideLabel attribute set to the value of true.

Sort Attribute

Custom Tag

<E2:Table data="<%= selectableData %>" id="hideLableTable">
    <E2:TableColumn dataType="alpha" label="First Name" property="fname" sort="true"  />
    <E2:TableColumn dataType="alpha" label="Last Name" property="lname" sort="true" />
    <E2:TableColumn dataType="alpha" label="Address" property="address" sort="true" />
    <E2:TableColumn dataType="alpha" label="Address Issue" property="address" style="min-width" hideLabel="true">
        <E2:Notifier />
    </E2:TableColumn>
</E2:Table>

The sort attribute is used to identify what columns developer want end users to have the ability to on-click sort data. This only works on with contents loaded on the screen.

Sample JSON

{
    "template": "table",
    "caption": "Generic Table",
    "head": {
        "rows": [
            {
                "columns": [
                    {
                        "type": "header",
                        "text": "First Name",
                        "attributes": {
                            "data-type": "alpha"
                        },
                        "sort": true
                    },
                    {
                        "type": "header",
                        "text": "Last Name",
                        "attributes": {
                            "data-type": "alpha"
                        },
                        "sort": true
                    },
                    {
                        "type": "header",
                        "text": "Address",
                        "attributes": {
                            "data-type": "alpha"
                        },
                        "sort": true
                    },
                ]
            }
        ]
    },
    "body": {
        ... Omitted Body Content ...
    }
}

Sort is a simple attribute flag. By adding the attribute to the custom tag with a value of true, will cause the property to be included on the render JSON. As shown in the example above, this will only add the sort property to the table header columns that contain the corresponding attribute.

This functionality is still being worked on. The example will be updated once the scripts have been finished!

Aggregate Attribute

Custom Tag

<E2:Table data="<%= selectableData %>" id="hideLableTable">
    <E2:TableColumn dataType="alpha" label="City" property="fname" />
    <E2:TableColumn dataType="alpha" label="Country" property="lname" />
    <E2:TableColumn dataType="alpha" label="Population" property="address" aggregate="sum" format="quality" />
</E2:Table>

When using the aggregate function, you are really creating a table footer (tfoot) section of a table. By default for each aggregate column use the tfoot will contain 1 table footer column.

Developers can also use the footerLabel and footerInitialSpan attributes on the table header to add special footer label column. For more information on these attributes see the E2:Table documentation.

Sample JSON

"footer": {
    "rows": [
        {
            "columns": [
                {
                    ... Omitted footerLabel example ...
                },
                {
                    "text": "1,000,000"
                }
            ]
        }
    ]
}

When turned into JSON, the table footer section appears much like every other table section. There are rows and column breakdowns. But in the example above, the last column of the only row is what the aggregate function created. Aggregate rows look just like regular column. Since the values are simply static text the value render inside of the text property. Through it is important to note, if a column has any special style attribute applied to it, the style attribute will be passed along to the aggregate column as well.

Button Columns

Custom Tag

<E2:Table data="<%= selectableData %>" id="hideLableTable">
    <E2:TableColumn dataType="alpha" label="City" property="fname" />
    <E2:TableColumn dataType="alpha" label="Country" property="lname" />
    <E2:TableColumn dataType="alpha" label="Population" property="lname" />
    <E2:TableColumn dataType="alpha" label="Edit">
        <E2:Button viewKey="BTN_EDIT" eventScript={} />
    </E2:TableColumn>
</E2:Table>

In the past developers needed to create an assign button definitons directly on the table. Going forward table buttons will now appear inside of the table directly using and developers declaring event scripts and apply attributes that will use new global functions.

Sample JSON

{
    "columns": [
        {
            "text": "Albany"
        },
        {
            "text": "Albany"
        },
        {
            "text": "500,000"
        },
        {
            "text": "0.9",
            "visibility": "hidden"
        },
        {
            "contents": [
                {
                    "type": "button",
                    "template": "field",
                    "input": {
                        "attributes": {
                            "id": "btn_edit_1",
                            "onclick": "emp.functionCall(event, 'emp.processMap', [{'E_CITY':0, 'E_COUNTY':1, 'E_POPULATION':2, 'E_TAXRATE':3 }, ])",
                        },
                        "text": "Edit"
                    }
                }
            ]
        }
    ]
}

When any E2:TableColumn contains another E2 tag, the contents of that tag should be placed inside of a contents array. In the example above, we create a button that should appear inside of this row. When this happens the normal button definition is process and its contents are added as a sub-object to the column contents array.
