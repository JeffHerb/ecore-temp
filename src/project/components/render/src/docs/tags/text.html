Text Custom Tag

The <E2:Text> creates a standard text field. For special input types (radio, checkbox, select, or textarea) see their specific documentation pages.

Examples

Custom Tag

<E2:Text control="text" viewKey="demo" />

Sample JSON

{
    "type": "text",
    "template": "field",
    "label": {
        "text": "Demo Text Label",
        "attributes": {
            "for": "demo"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo",
            "type": "text",
            "value": "",
            "data-validation": "test1,test2,test3"
        }
    }
}

The viewKey attribute must be specified. The view key will drive most of the field's characteristics — usually its label, size, validation, and more.

The <E2:Text> custom tag supports many different types of HTML 5 inputs. Developers can set the control tag to any of the following:

  • text
  • email
  • month
  • number
  • range
  • password
  • search
  • tel
  • time
  • url
  • week
  • color

If the control attribute is missing the tag will default to text. Note that this custom tag only creates the field — additional layout and structure should be defined by parent JSP tags (e.g. <E2:Row> and <E2:Col>). Also, not all of the different types of inputs are listed here; elements requiring different markup, such as check boxes and radio buttons, have their own specific custom tags.

Default example

For the <E2:Text> custom tag the JSON will actually generate information for two separate HTML elements: the <input> itself and its corrosponding <label> element. Based on the viewKey and other attributes, different JSON properies will be generated as well.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
viewKey String Yes A registered view key (previously called pageElementName)
type String No Defines the input's control type. See the acceptable type list above or use one of the more specific custom tags. Will default to text is nothing is provided.
value String No Initial value of the input
required Boolean No Indicates whether the field must be filled in (more details)
readOnly Boolean No Indicates whether the input should be read-only or updatable mode (more details)
title String No Sets the title attribute of the input element.
size Integer No Set the size attribute of the input element.
style String No Add specific style to a element or a group of elements (more details).
eventScript String No Any event-based JavaScript that needs to be executed. The string must include both event type and function name (more details)
accessKey String No Shortcut key to set focus to the field (more details)
visibility String No Used to control the visibility state of a particular input and/or its label (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE
  • HIDE_LABEL
  • HIDE_LABEL_NO_SPACE
  • HIDE_LABEL_YES_SPACE
inputHtmlAttributes String No Adds HTML attributes to the rendered <input> tags (more details)
labelHtmlAttributes String No Adds HTML attributes to the rendered <label> tags (more details)
property Object No Java object from which the value is populated. It is the responsibility of the custom tag or its parent custom tag to make the object available to the page context.
rsMapping String No Used to map this field to result set. Please see the ResultSet custom tag documentation for details.
message hashmap No Provides a hashmap of a specific field level message to an input (more details).
dataType string No Defined the type of data that will pre presented in the text box. Currently the only supported types are:
  • alpha (defualt)
  • date (recommended to use E2:Date instead)
  • numeric
format string No Changes the a inputs value to match a specific structure (more details).
