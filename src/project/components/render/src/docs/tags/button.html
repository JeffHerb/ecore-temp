Button Custom Tag

Button controls for submitting forms and providing interactive components. This tag collapse the following tags:

  • ET:AjaxButton
  • ET:PrimaryButton
  • ET:TabNavButton
  • ET:JSButton

Examples

Custom Tag

<E2:Button viewKey="button" />

Sample JSON

{
    "type": "button",
    "template": "field",
    "input": {
        "attributes": {
            "id": "button",
            "name": "button",
            "type": "button"
        },
        "text": "Button"
    }
}

<E2:Button> are form controls that kick off specific interactions or transactions.

Buttons generally follow the same JSON structure as other input types. The only exceptions are that they do not have labels and the value attribute is not required.

As with other input types, all attributes for a button should be defined in the attributes object. However, the text (visual label) should be set using its own text property inside the input object.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
viewKey String Yes Registered view key (previously called buttonTagId)
text String No The text at should appear inside of the button.
title String No Sets the title attribute (hover text) for the button
preSubmitScript String No Name and arguments to a function that is execute "onclick", but before the form has been submitted (more details)
visibility String No Controls the visibility state of the button (more details)
Acceptable Options:
  • HIDE_ON_READ
  • HIDE_ON_UPDATE
primary Boolean No Applies "primary" styling to the button
htmlAttributes String No Adds HTML attributes to the rendered button element (more details)
eventScript String No List of event-based JavaScript associated with the field. The string must include both the event type and function name (more details)
accessKey String No Shortcut key to set focus to the field (more details)
type String No Used to control the type attribute this button will render as. Buttons can be either set to button or submit. The default value is button

Buttons without viewKey Examples

Custom Tag

<E2:Button text="Interactive Button" id="inter-button" eventScript="{...script omitted ...}"/>

Primary JSON

{
    "type": "button",
    "template": "field",
    "input": {
        "attributes": {
            "id": "inter-button",
            "type": "button"
        },
        "text": "Interactive Button"
    }
}

When developers need an interactive button control, simply exclude the viewKey. When the button renders in JSON it will appear sligntly different as the addtiional attributes property of type. This will make it so the button does not automatically submit a form on click. If developers need to submit the form this button appears in or another form on click, they should use a viewKey based button or add the JavaScript required to their event script to kick off the action.

Primary Examples

Custom Tag

<E2:Button viewKey="button" primary="true" />

Primary JSON

{
    "type": "button",
    "template": "field",
    "input": {
        "attributes": {
            "id": "button1",
            "name": "button1",
            "type": "button"
        },
        "text": "Primary Button",
        "primary": true
    }
}

Set the primary attribute to true to create a primary button.

Icon Examples

Custom Tag

<E2:Button viewKey="button" primary="true" />

Icon JSON

{
    "type": "button",
    "template": "field",
    "input": {
        "attributes": {
            "id": "button1",
            "name": "button1",
            "type": "button"
        },
        "icon": "printer"
        "text": "Print Page"
    }
}

To render a button as an icon, rather than text, provide a string value for the name of the icon. In the example above we are adding the printer icon, so we used the value print. Icon names are predefined and must be registered.

Control attribute

Custom Tag

<E2:Button viewKey="button" control="submit" />

Icon JSON

{
    "type": "button",
    "template": "field",
    "input": {
        "attributes": {
            "id": "button1",
            "name": "button1",
            "type": "submit"
        },
        "icon": "printer"
        "text": "Print Page"
    }
}

By default when buttons render they render as a standard button control; or a button with the attribute of type set to button. This is fine if a button is being used for JavaScript interactions, but if the button is used to control a form, then it needs to be turned into a submitem> button. To make the change, use the control attribute with the value of submit.
