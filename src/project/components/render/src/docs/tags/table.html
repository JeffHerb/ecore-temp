Tables Custom Tag

In Review

The <E2:Table> tag is used to create HTML tables (formerly known as result sets).

Examples

Custom Tag

<E2:Table data="<%= bodyArray %>" id="table-id" />
    <E2:TableColumn label="skipped" skip="true" />
    <E2:TableColumn label="Column1" />
    <E2:TableColumn label="Column2" />
    <E2:TableColumn label="Column3" />
    <E2:TableColumn label="Column4" />
    <E2:TableColumn label="Column5" />
</E2:Table>

Sample JSON

{
    "template": "table",
    "attributes": {
        id: "table-name"
        "data-displayRows": "10"
    },
    "head": {
        "rows": [
            {
                "columns": [ .. Data Hidden .. ]
            }
        ]
    },
    "body": {
        "rows": [
            {
                "columns": [ .. Data Hidden .. ]
            },
            {
                "columns": [ .. Data Hidden .. ]
            },
            {
                "columns": [ .. Data Hidden .. ]
            }
        ]
    }
}

To create a basic table, developers must define the E2:Table tag and then provide each table with a unique id and the data attribute used to a specific collection of data they wish to display. After that, developers need to define the internal column require. For more information on table columns see the E2:TableColumn documentation.

When looking at the tables JSON markup, it is important to look at the outer most object and properties. Simlilar to other E2 tags, tables include the template property and will always have the value of table. Table specific attributes will appear in the attributes object. Three additional section objects (head, body, and footer) can also appear in the root of the table JSON.These represent the key literal thead, tbody, and tfoot and will in the JSON as they are required. 

"head": {
    "rows": [
        {
            "columns": [
                {
                    ... Contents Ommitted ...
                },
                {
                    ... Contents Ommitted ...
                }
            ]
        }
    ]
},

When looking at a section structure there are a few key things to point out. First all table sections, head, body, and footer footer are rendered as objects. This section naming is required to ensure proper display structure. Next every section will have a rows propery, that will contain an array of object representing each row. Each object row will then contain its own object of columns which is another array of object representing each column of the table. Depending on the section, configuration and contents, this object will vary.

For additional levels of functionality control see the example code and table level attributes below.

Custom Tag Attributes
Attribute Name Type Required Purpose
data String Yes Java hashmap or arrayList used to popuplate the table contents.
id String Yes The desired id for the table This will also turn into the html ID of the table..
title String No The displayed title or table caption.

The table title attribute is how developers add table captions or titles. When this is rendered into JSON, it will appear as a root property on the top of the table. In The HTML the title is rendered via the caption element.

Sticky Header, Sticky Footers Example

Custom Tag

<E2:Table data="<%= bodyArray %>" id="sticky-example" stickyHeader="true" />
    <E2:TableColumn label="Index" />
    <E2:TableColumn label="Column1" />
    <E2:TableColumn label="Column2" />
    <E2:TableColumn label="Column3" />
    <E2:TableColumn label="Column4" />
    <E2:TableColumn label="Column5" />
</E2:Table>

Sample JSON

{
    "template": "table",
    "caption": "Display Rows Table",
    "attributes": {
        "id": "sticky-header",
        "data-sticky": "header"
    },
    "head": { ... Omitted ... },
    "body": { ... Omitted ... }
}

Sticky headers and footer can be easily added by simply defining the attribute of stickyHeader or stickyFooter to the JSP tag with the value set to true.

When this JSP attribute is rendered into JSON, it will add the attribute of data-sticky to the table element. In the example above the value is set to just header as it is the only item being requested. If both header and footer were defined as stick they would appear as a comma seperated value, e.g. "header,footer".

Selectable Rows Example

Custom Tag

<E2:Table data="<%= bodyArray %>" selectable="single" id="single-selection" />
    <E2:TableColumn label="First" />
    <E2:TableColumn label="Last" />
    <E2:TableColumn label="Middle" />
</E2:Table>

Sample JSON

{
    "template": "table",
    "title": "Table Title",
    "attributes": {
        "id": "single-selection"
    },
    "selectable": true,
    "selectionType": "single",
    "head": {
        "rows": [
            {
                "columns": [
                    {
                      ... Ommitted remaining header columns ...
                    }
                ]
            }
        ]
    },
    "body": {
        "rows": [
            {
                "columns": [
                    {
                        "text": "Jeff"
                    },
                    {
                        "text": "Herb"
                    },
                    {
                        "text": "A"
                    }
                ]
            },
            {
                ... Omitted remaining rows ...
            }
        ]
    }
}

The selectable attribute is used to indicate when a table should render with the additional first column containing either a radio or checkbox control. The developer does not need to write any additional table markup to reference the control column. To toggle between radio or checkbox developer must set the selectable tag to either single or multiple.

In this particular case, there is no additional JSON added at the individual row level. Instead two additional attribute should appear in the root of the table object selectable and selection. The selectable property is used to indicate that the table needs the additional columns. The selectionType is used to determine if the additional column should contain either a radio or checkbox

It is also important to note that the radio/checkbox control name and id attributes are pre-defined. In the example above, all radios will share the same name tag. The value of the name tag is the same as the table id. But since ids need to differ, the id of the radio has the suffixed to include its row number. In this case the suffix is _1. All indexs should follow the underscore number pattern for as many rows the table contains on the screen. If this table was setup to be multiple select, then the name and the id would both be suffix with the row index number.

Select All Rows Example

Custom Tag

<E2:Table data="<%= bodyArray %>" id="multiple-selection" selectable="multiple" selectAll="true" />
    <E2:TableColumn label="First" />
    <E2:TableColumn label="Last" />
    <E2:TableColumn label="Middle" />
</E2:Table>

Sample JSON

{
    "template": "table",
    "title": "Multi Selection Table",
    "attributes": {
        "id": "multi-selection"
    },
    "selectable": true,
    "selectionType": "multiple",
    "selectAll": true,
}

By default selectable rows only affects the rows in the body of the table. But if the table requires a "select all" checkbox at the top, then the additional attribute selectAll needs to be added and have the value set to true. It should be noted that this feature should only be used on tables with multiple (checkbox) selections.

Note in the JSON is slightly changes to include the selectAll property as part of the tables root object.

FooterLabel and FooterInitalSpan Example

Custom Tag

<E2:Table data="<%= bodyArray %>" id="footer-example" footerLabel="Grand Total" footerInitialSpan="2" />
    </E2:TableColumn label="City" property="city" />
    </E2:TableColumn label="County" property="country" />
    </E2:TableColumn label="Populations" property="population" format="Number" />
</E2:Table>

Sample JSON

"footer": {
    "rows": [
        {
            "columns": [
                {
                    "span": 2,
                    "text": "Grand Total",
                    "footerLabel": true
                },
                {
                    "text": "1,000,000"
                }
            ]
        }
    ]
}

Table footers are created when individual columns are marked with than aggregate function. In order to provide a label in the table, developers can define the additional attribute of footerLabel on the JSP. With this additional attribute present a new leading footer column in the footer row definition. Developers can also define the additional attribute of footerInitialSpan. This attribute takes a number, this number representing the number of column the new leading column should span.

When this is rendered into JSON, the table now has two columns defined in the footer section. The first is our new footer label column. First, every footer label column will contain the property of footerLabel set to the value of true. This is simply a flag for our render so we can add style hooks. The columns should then contain the text property which should be the string value provided in the JSP footerLabel attribute. Since footerInitialSpan is defined on the JSP, the span property also appears and is set to the number provided on the tag.

Sample JSON

At times rows need to have an additional indentifier to connect serverside data to the client browser. The best way to do this is to add the keyProperty attribute on the E2:Table tag. The keyProperty attribute provides developers with to prefix a set of data or define a row value that will uniquely identify this particular row.

When rendered into JSON, this key value will appear as a property inside of every row object as seen above. When turned into HTML the key value will appear on every tr row element as the attribute data-rowkey. If the table is setup to have selectable rows, the radio/checkboxes will also us the row key as their values.

Pagination Attirubutes: AJAX URL and Page Amount

Custom Tag

<E2:Table data="<%= bodyArray %>" id="paging-table" ajaxURL="http://www.google.com" pageAmount="25" />
    <E2:TableColumn label="skipped" />
    <E2:TableColumn label="Column1" />
    <E2:TableColumn label="Column2" />
    <E2:TableColumn label="Column3" />
    <E2:TableColumn label="Column4" />
    <E2:TableColumn label="Column5" />
</E2:Table>

Sample JSON

{
    "template": "table",
    "caption": "Display Rows Table",
    "attributes": {
        "id": "paging-table",
        "data-ajax": "http://www.google.com",
        "data-page-amount": 25,
        "data-total-rows": 100
    },
    "head": { ... Omitted ... },
    "body": { ... Omitted ... }
}

By default tables will not paginate unless a developer defines the URL in which the additional data can be collected from. In order to provide this, developer need to give a value URL using the ajaxURL attribute. In the past developers needed to create buttons and register them with the Registration tool. Going forward this additional buttons will be created for developers on the client side so they no longer need to be registered. Developers can also control the amount of records being returned per page using the pageAmount attribute. It is also important to note that this same ajax URL will be the same URL used to control server side sorting and filtering of tables (more information coming later).

When this is changed to JSON the table object will pick up some additional attributes. data-ajax will be the URL provided by ajaxURL attribute. If ajaxURL is defined, the data-page-amount attribute will also appear. The value with default to 50 unless the developer used the pageAmount attribute. There should also be a data-total-rows, to indicate how many rows a particular table has.
