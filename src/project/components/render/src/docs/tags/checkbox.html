CheckBox Custom Tag

The <E2:CheckBox> tag is used to create individual checkbox elements.

Examples

Custom Tag

<E2:CheckBox viewKey="demo" value="y" />

Sample JSON

{
    "type": "checkbox",
    "template": "field",
    "label": {
        "text": "Checkbox Label",
        "attributes": {
            "for": "E_CHECK_1"
        }
    },
    "input": {
        "attributes": {
            "name": "E_CHECK_1",
            "id": "E_CHECK_1",
            "type": "checkbox",
            "value": "y"
        }
    }
}

CheckBox JSON objects are structured like every other input but with the type attribute set to checkbox.

Rendered examples

Just like other inputs, all attributes are tied to the viewKey. As the example shows, check boxes objects include the label object first followed by input. This example shows an individual check box, however check boxes normally appear in groups — see the CheckBoxGroup tag for details on working with groups.

Flip Examples

Custom Tag

<E2:CheckBox viewKey="demo" value="y" flip="true" />

Flip JSON

{
    "type": "checkbox",
    "template": "field",
    "label": {
        "text": "Checkbox Label",
        "attributes": {
            "for": "E_CHECK_2"
        }
    },
    "input": {
        "attributes": {
            "name": "E_CHECK_2",
            "id": "E_CHECK_2",
            "type": "checkbox",
            "value": "true"
        },
        "flip": true
    }
}

By adding the flip attribute to the JSP tag, an additional flip property with the value of true should be added to the input object.

Rendered examples

Checked Examples

Custom Tag

<E2:CheckBox viewKey="demo" value="y" checked="true" />

Checked JSON

{
    "type": "checkbox",
    "template": "field",
    "label": {
        "text": "Checkbox Label",
        "attributes": {
            "for": "demo_1"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo_1",
            "type": "checkbox",
            "value": "true",
            "checked": true
        }
    }
}

When the checked attribute is added to the JSP it will set the checked attribute to the input's attribute list in the JSON.

ReadOnly Examples

Custom Tag

<E2:CheckBox viewKey="demo" value="y" readOnly="true" icon="Checkmark" />

Checked JSON

{
    "type": "checkbox",
    "template": "field",
    "label": {
        "text": "Checkbox Label",
        "attributes": {
            "for": "demo_1"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo_1",
            "type": "checkbox",
            "checked":true
        }
        "requestedMode": "1",
        "icon": "not-applicable"
    }
}

Requested mode for checkboxes are similar to other inputs with one exception. If the developer forces an input into a read-only state, they can specify an additional attribute of icon. This will allow the developer to swap the check box's text value with a single image. See the <>. This adds the additional input sub-property of icon to the JSON.
