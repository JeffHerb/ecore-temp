Radio Custom Tag

The Radio tag is used to create individual radio elements.

Examples

Custom Tag

<E2:Radio viewKey="demo" value="y"/>

Sample JSON

{
    "type": "radio",
    "template": "field",
    "label": {
        "text": "Radio Label",
        "attributes": {
            "for": "demo_1"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo_1",
            "type": "radio",
            "value": "y"
        }
    }
}

Radio tags JSON look like every other pre-specified of input. The difference here being that the type attribute is set to radio.

Rendered examples

Just like other inputs, all attributes are tied to the viewKeys. As the example shows, radio buttons appear label tag first then input. Unlike other, radio normally appear in groups, this example is simply to show individual radio tags. Reference the RadioGroup tag for more specifics on working with groups.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
value String Yes Default value of the input.
check String No Indicate if the input is checked on load.
title String No Sets the title attribute of the input element.
accessKey String No Shortcut key to set focus to the field (more details)
eventScript String No Any event-based JavaScript that needs to be executed. The string must include both event type and function name (more details)
labelHtmlAttributes String No Adds HTML attributes to the rendered <label> tags (more details)
inputHtmlAttributes String No Adds HTML attributes to the rendered <input> tags (more details)
property Object No Java object attribute from which value is populated. It is the responsibility of the tag or its parent tag to make the object available to the page context.

Flip Examples

Custom Tag

<E2:Radio viewKey="demo" value="y" flip="true" />

Flip JSON

{
    "type": "radio",
    "template": "field",
    "label": {
        "text": "Radio Label",
        "attributes": {
            "for": "demo_1"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo_1",
            "type": "radio",
            "value": "y"
        },
        "flip": true
    }
}

By adding the flip attribute to the JSP tag, an additional flip property with the value of true should be added to the input object.

Check Examples

Custom Tag

<E2:Radio viewKey="demo" value="y" checked="true" />

Checked JSON

{
    "type": "radio",
    "template": "field",
    "label": {
        "text": "Radio Label",
        "attributes": {
            "for": "demo_1"
        }
    },
    "input": {
        "attributes": {
            "name": "demo",
            "id": "demo_1",
            "type": "radio",
            "value": "y",
            "checked": true
        }
    }
}

When the checked attribute is added to the JSP, this will set the checked attribute to the inputs attribute list.
