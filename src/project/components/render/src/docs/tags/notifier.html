Notification Tag

The <E2:Notifier> is used to create notifiers.

Examples

Custom Tag

<E2:Notifier notifierID="noteStatic">

Static JSON

{
    "type": "static",
    "template": "notifier",
    "attributes": {
        "alt": "Static Notifier",
        "id": "noteStatic"
    }
    "text": "SN"
}

<E2:Notifier> tags are driven by their notifierID which is required. That ID comes from the notifier reference table and determines what type of a notifier to display.

Static Rendered Examples

Notifiers that do not contain a reference to a specific pageId will render as a static notifier. This means the content of the notifier is rendered in a <span> element. Note that the attribute id should match the requested notifierID attribute.

Examples

Custom Tag

<E2:Notifier notifierID="note1">

Interactive JSON

{
    "type": "interactive",
    "template": "notifier",
    "attributes": {
        "alt": "Interactive Notifier",
        "href": "internalpage.jsp",
        "id": "noteInteractive"
    },
    "text": "IN"
}

When a <E2:Notifier> reference includes a pageID it will render as an <a> element. An additional href attribute needs to be included in the JSON as depicted above.

Options

Custom Tag Attributes
Attribute Name Type Required Purpose
notifierID String Yes The notifier ID used in the NOTF reference table.
preSubmitScript String No Name and arguments to a function that is execute "onclick", but before the form has been submitted (more details)
eventScript String No Any event-based JavaScript that needs to be executed. The string must include both event type and function name (more details)
htmlAttributes String No Adds HTML attributes to the rendered <span> or <a> element (more details)
